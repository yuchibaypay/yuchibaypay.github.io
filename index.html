    Introduction

    Welcome to the BayPay API!

    BASE URL :

    https://api.baypay.com

    Authentication

    Client need to ask for an appId and HashKey to allow access to the API.

    Account

    CheckToken

    Check if SessionToken is vaild

    REQUEST BODY

    {
  "pushToken": "",
  "os": "",
  "appVersion": "",
  "appCode": "",
  "sessionToken": "ea6273df-c516-4f6d-a713-b85ad0a6d375"
}

    RESPONSE BODY

    {
  "returnCode": 1,
  "message": "",
  "result": {
    "isShop": false,
    "sessionToken": "c19233a2-1a17-4089-b7f9-74ef4de08a0a"
  }
}

    HTTP Request

    POST /api/Account/checkToken

    Request Headers

    Name Value
    Content-Type application/json

    Request Body

    Parameter Type Required Description
    pushToken string N 使用推播需要傳入的token
    os string N os 版本
    appVersion string N app 版本
    appCode string N app 代號
    sessionToken string Y sessionToken

    Response Body

    Attribute Type Description
    returnCode string ReturnCode
    message string message
    result object response data

    result

    Attribute Type Description
    isShop bool 角色是否為商家
    sessionToken string 如果傳入的 sessionToken 是正確, 會回傳新的 sessionToken

    Register

    Register an account

    REQUEST BODY

    {
  "code": "a86db9cfb6b61c6866347acba16940a0",
  "appId": 9,
  "phone": "+886933035648",
  "password": "WVkfkrHp9qCekY7RJVy/GOGiobVWWfk/w4RZeIysXzg=",
  "time": 1601266526
}

    RESPONSE BODY

    {
  "returnCode": 4,
  "message": "電話尚未認證",
  "result": null
}

    HTTP Request

    POST /api/Account/register

    Request Headers

    Name Value
    Content-Type application/json

    Request Body

    Parameter Type Required Description
    code string Y MD5Hash產生 (appId + phone + password + time + hashKey)
    appId number Y appId (測試使用9)
    phone string Y 有加國碼的手機號碼, 例:+886987654321
    password string Y 密碼
    time number Y unix timestamp

    Response Body

    Attribute Type Description
    returnCode string ReturnCode
    message string message
    result object response data

    POST /api/Account/login

    Request Headers

    Name Value
    Content-Type application/json

    Request Body

    Parameter Type Required Description

    seesionToken string 每次登入都會產生新的seesionToken

    Errors

    This error section is stored in a separate file in includes/_errors.md. Slate allows you to optionally separate out your docs into many files...just save them to the includes folder and add them to the top of your index.md's frontmatter. Files are included in the order listed.

    The Kittn API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The kitten requested is hidden for administrators only.
    404 Not Found -- The specified kitten could not be found.
    405 Method Not Allowed -- You tried to access a kitten with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The kitten requested has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- You're requesting too many kittens! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
